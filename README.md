## ✨ Возможности

### 🌐 Веб-интерфейс (Streamlit)
- **Календарь матчей** — поиск матчей за период с фильтрацией по статусу
- **Парсинг матча** — детальная статистика игроков и команд
- **Парсинг турнира** — турнирная таблица и шахматка результатов
- **Экспорт данных** — скачивание результатов в CSV

### 🖥️ CLI-утилиты
- Интерактивный выбор сезона и этапа турнира
- Парсинг отдельного матча по ID
- Поиск матчей за указанный период

## 🚀 Использование

### Веб-интерфейс (рекомендуется)

```bash
# Запуск Streamlit-приложения
streamlit run app.py
```

После запуска откроется браузер с адресом `http://localhost:8501`

**Навигация по приложению:**
| Вкладка | Описание |
|---------|----------|
| 📅 Календарь | Поиск матчей за период, выбор матча для парсинга |
| 🏀 Матч | Детальный парсинг статистики по ID матча |
| 📊 Турнир | Парсинг турнирной таблицы и шахматки |
| ℹ️ О проекте | Информация об API и форматах данных |

### CLI-режим

#### 1. Парсинг турнира (standings + crosstable)

```bash
python main.py
```

**Алгоритм работы:**
1. Ввести ID сезона (например, `50788`)
2. Выбрать номер этапа из списка
3. Получить файлы: `standings_<ID>_<этап>.csv` и `crosstable_<ID>_<этап>.csv`

#### 2. Парсинг отдельного матча

```bash
python match_parser.py <GAME_ID>
```

**Пример:**
```bash
python match_parser.py 1016417
```

**На выходе:**
- `match_<ID>_players.csv` — статистика игроков
- `match_<ID>_teams_total.csv` — статистика команд

#### 3. Поиск матчей за период

```bash
python match_finder.py [COMP_ID]
```

**Пример:**
```bash
python match_finder.py 50758
```

**На выходе:** `matches_list_<COMP_ID>.csv` со списком матчей

## 🌐 API Endpoints

| Endpoint | Назначение | Пример |
|----------|------------|--------|
| `Widget/CompIssue/{id}` | Список этапов сезона | `?id=50788` |
| `Widget/CompTeamResults/{id}` | Турнирная таблица (standings) | `?id=50758` |
| `Widget/CrossTable/{id}` | Результаты матчей (crosstable) | `?id=50758` |
| `Comp/GetCalendarCarousel/` | Календарь матчей по периоду | `?comps={id}&from=today-14&to=today+14` |
| `Widget/GetOnline/{game_id}` | Протокол матча (составы, статистика) | `?game_id=1016417` |
| `Api/GetOnlinePlays/{game_id}` | Play-by-play лог событий | `?last=0` |

**Headers для всех запросов:**
```python
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ...',
    'Referer': 'https://belarus.russiabasket.ru/'
}
```

## 📊 Форматы выходных данных

### standings_*.csv — Турнирная таблица

| Колонка | Описание |
|---------|----------|
| Место | Позиция в таблице |
| Команда | Название + город |
| И | Игры сыграно |
| В | Победы |
| П | Поражения |
| Забито | Очков забито |
| Пропущено | Очков пропущено |
| Очки | Турнирные очки (2 за победу, 1 за поражение) |
| +/- | Разница забитых/пропущенных |
| % | Процент побед |

### crosstable_*.csv — Шахматка результатов

Матрица результатов между командами:
- Формат: `Счёт` + `д` (дома) / `г` (в гостях)
- Для будущих матчей: `(дата)`
- Диагональ: `---`

**Пример:**
```
,Минск,Борисов,Гомель
Минск,---,85:72 д,90:88 г
Борисов,72:85 г,---,78:80 д
Гомель,88:90 д,80:78 г,---
```

### match_*_players.csv — Статистика игроков

| Колонка | Описание |
|---------|----------|
| Команда, Имя, №, Роль | Информация об игроке |
| Мин | Игровое время (MM:SS) |
| Очки, 2-очк, 3-очк, ШБ | Броски (попадания/попытки) |
| Подборы (O/D/T) | Подборы (атака/защита/всего) |
| ГП, ПХ, ПТ, БШ, Ф, ФС | Голевые, перехваты, потери, блокшоты, фолы, фолы на игроке |
| +/- | Плюс/минус |
| КПИ | Коэффициент полезности |

### match_*_teams_total.csv — Статистика команд

Суммарная статистика по команде за матч (аналогично игрокам).

### matches_list_*.csv — Список матчей

| Колонка | Описание |
|---------|----------|
| ID матча | Уникальный идентификатор |
| Дата | Дата проведения |
| День | День недели |
| Хозяева / Гости | Названия команд |
| Счет/Время | Счёт (если завершён) или время начала |
| Статус | ЗАВЕРШЕН / LIVE / ЗАПЛАНИРОВАН |
| Турнир | Название соревнования |

## 🛠️ Конвенции разработки

### Code Style
- **Язык комментариев:** Русский
- **Именование:** `snake_case` для функций и переменных
- **Обработка ошибок:** Try-except блоки с выводом в консоль
- **Вспомогательные функции:** `safe_int()`, `safe_split_shots()` для защиты от некорректных данных

### Architecture Patterns
- **Модульность:** Отдельные файлы для разных типов парсинга
- **CLI-интерфейс:** Интерактивный ввод через `input()`
- **Экспорт:** CSV с кодировкой `utf-8-sig` (корректное отображение в Excel)

### Расчёт КПИ (Коэффициент Полезности)
```
КПИ = (Очки + Подборы + Ассисты + Перехваты + Блокшоты + Фолы на) 
      - (Промахи с игры + Промахи со ШБ + Потери + Фолы)
```

## ⚠️ Известные проблемы

1. **API нестабильно** — структура может измениться (неофициальное API)
2. **Два режима парсинга матчей:**
   - **Live/свежие:** реконструкция из play-by-play лога (`parse_from_log`)
   - **Архивные:** готовые данные из JSON (`parse_from_final_json`)
3. **Молодёжная статистика** — может встречаться в данных, пропускается в шахматке
4. **Отсутствие тестов** — верификация через ручной запуск и проверку CSV-файлов
