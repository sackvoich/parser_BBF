# Basketball Parser BBF

Парсер статистики белорусского баскетбола для сайта `belarus.russiabasket.ru`.

## Обзор проекта

Python-скрипт для автоматического сбора и обработки баскетбольной статистики из API `org.infobasket.su`. Проект извлекает данные о матчах, формирует турнирные таблицы и шахматки результатов.

### Компоненты

| Файл | Назначение |
|------|------------|
| `main.py` | CLI-интерфейс: ввод ID сезона, выбор этапа турнира |
| `stats_parser.py` | Расчёт турнирной таблицы (standings) и шахматки (crosstable) |
| `parser_v3.py` | Детальный парсинг статистики игроков по отдельным матчам |

### Технологии

- **Python 3.14** (судя по `pyvenv.cfg`)
- **pandas** — обработка данных, CSV-экспорт
- **requests** — HTTP-запросы к API
- **numpy** — вычисления (через pandas)

## Установка и запуск

### 1. Активация виртуального окружения

```bash
# Windows
venv\Scripts\activate

# Linux/macOS
source venv/bin/activate
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Запуск парсера

```bash
python main.py
```

**Рабочий процесс:**
1. Ввести ID сезона (например, `50788`)
2. Выбрать номер этапа из предложенного списка
3. Получить CSV-файлы: `standings_<ID>_<этап>.csv` и `crosstable_<ID>_<этап>.csv`

### 4. Парсинг отдельного матча

```bash
python parser_v3.py <GAME_ID>
```

Пример:
```bash
python parser_v3.py 1016417
```
Создаст файл `match_1016417_stats.csv` со статистикой игроков.

## Структура выходных данных

### standings_*.csv (Турнирная таблица)

| Поле | Описание |
|------|----------|
| Место | Позиция в таблице |
| Команда | Название + город |
| И | Игры сыграно |
| В | Победы |
| П | Поражения |
| Забито | Очков забито |
| Пропущено | Очков пропущено |
| Очки | Турнирные очки (2 за победу, 1 за поражение) |
| +/- | Разница забитых/пропущенных |
| % | Процент побед |

### crosstable_*.csv (Шахматка)

Матрица результатов между командами:
- Формат: `Счёт` + `д` (дома) / `г` (в гостях)
- Для будущих матчей: `(дата)`
- Диагональ: `---`

### match_*.csv (Статистика матча)

| Поле | Описание |
|------|----------|
| Команда | Название команды |
| Игрок | ФИО игрока |
| № | Номер на форме |
| Роль | Игрок / Тренер / TEAM |
| Мин | Игровое время |
| Очки | Набранные очки |
| 2-очк, 3-очк | Точность бросков |
| СЩ, ЧЩ, ВС | Подборы (защитные, чужие, всего) |
| ГП, ПХ, ПТ, БШ, Ф | Голевые, перехваты, потери, блокшоты, фолы |
| +/- | Плюс/минус |
| КПИ | Коэффициент полезности |

## API Endpoints

| Endpoint | Описание |
|----------|----------|
| `Widget/CompIssue/{id}` | Список этапов сезона |
| `Widget/CompTeamResults/{id}` | Результаты команд (для standings) |
| `Widget/CrossTable/{id}` | Результаты матчей (для crosstable) |
| `Widget/GetOnline/{game_id}` | Протокол матча |
| `Api/GetOnlinePlays/{game_id}` | Лог событий матча (play-by-play) |

## Особенности реализации

- **Два режима парсинга матчей:**
  - Live/свежие: реконструкция из лога событий (`parse_from_log`)
  - Архивные: готовые данные из JSON (`parse_from_final_json`)
- **Расчёт КПИ:** `(Очки + Подборы + Ассисты + Перехваты + Блокшоты) - (Промахи + Потери + Фолы)`
- **Плюс/минус:** считается динамически по времени на площадке

## Примечания

- Проект использует неофициальное API, структура может измениться
- Некоторые матчи могут не иметь лога событий (старые игры)
- В данных встречается молодежная/юношеская статистика (пропускается в шахматке)
